Rule proposal

Cách dùng tài liệu

Không đọc như một bài mô tả dài. Bắt đầu từ vai trò của bạn, xem visual demo, sau đó dùng bảng rule và checklist để tạo hoặc duyệt content.

A

Art / UI Artist

Dùng phần Sticker Spec, Grid Cell, Choice Card và Inline Sticker để thiết kế asset, safe area, scale và hierarchy.

Output cần bàn giao: sticker 1:1 transparent, canonical preview, small-size test và dual-cell preview.
G

Game Design

Dùng character budget, clue rule, canonical name và demo checker để viết level content không phá layout.

Output cần bàn giao: localized key, short name, clue copy, constraint và preview pass.
P

Product / PO

Dùng acceptance criteria và editor validation để chốt scope, tránh xử lý overflow thủ công sau khi content đã được sản xuất.

Quyết định cần chốt: font minimum, hard limit, locale support và có cho phép dual-value ở phase đầu hay không.

Clue measurement

1 item/icon = 2.5 Character Equivalent

Hiệu chỉnh từ screenshot Profile Perfect: footprint của inline icon lớn hơn rõ rệt một ký tự chữ thường. Vì asset và font có thể thay đổi, 2.5 CE là hệ số authoring an toàn, không phải phép quy đổi pixel tuyệt đối. Publish cuối vẫn dùng rendered width và số dòng thực tế.

abcdefg📰

2.5 CE dùng để lọc sớm content. Không dùng CE thay cho layout test: icon khác nhau có thể có optical width khác nhau, nên runtime/editor vẫn phải đo rendered width.

Formula

Công thức dùng cho GD & Editor

Effective CE = Text characters + (Inline item count × 2.5)

Label của item vẫn tính như text bình thường.
2.5 CE chỉ là phần footprint của icon.
48 text + 0 icon48 CE
46 text + 1 icon48.5 CE
44 text + 2 icon49 CE
48 text + 3 icon55.5 CE
Publish rule

Rule chốt

  • Soft budget: ≤ 48 CE.
  • Review: 48.5–60 CE, bắt buộc preview trên card thật.
  • Block: > 60 CE hoặc > 5 dòng.
  • Inline item: recommend ≤ 2; hard limit 3.
  • Icon + label phải là non-breaking token.

  • Mỗi value chỉ có 1 canonical name.

Clue Card Rule

Clue card có fixed height. Visual dưới đây cho GD thấy 5 dòng nằm ở đâu và cho Art/UI thấy inline token phải bám baseline như thế nào.

Rule content

  • Khuyến nghị ≤ 48 CE; hard limit 60 CE.
  • Effective CE = text characters + item/icon × 2.5.
  • Tối đa 5 dòng trên board 4–5 subject.
  • Một clue chứa 1 kết luận hoặc 1 quan hệ chính.
  • Tối đa 2 Inline Sticker khuyến nghị; hard 3.
  • Tối đa 3 màu highlight, không phụ thuộc màu để truyền meaning.
Không dùng: auto-scroll, marquee, ellipsis hoặc tăng riêng chiều cao card.

Advanced rule

Dual-value Cell

Dual-value là một mechanic riêng, không chỉ là “nhét thêm icon”. Visual dưới đây mô tả vùng của từng value và các trường hợp phải block.

Visual anatomy

🕶️🧛
  • Mỗi Sticker tối đa 40–44% cell.
  • Đường chéo có contrast thấp hơn border chính.
  • Không có label trong cell.
  • Mỗi nửa cell là một hit target logic riêng.

Interaction rule

  1. Chọn value thứ nhất → điền vào slot A.
  2. Tap “Add secondary” → popup chỉ show value hợp lệ.
  3. Chọn value thứ hai → điền slot B, không thay slot A.
  4. Tap từng nửa cell để xem/sửa value tương ứng.
Product decision: Thêm value thứ hai cần confirm hay tự động điền vùng trống?
